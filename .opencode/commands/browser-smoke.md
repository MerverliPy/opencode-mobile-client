Use the validator agent.

Goal:
Validate the current mobile shell with the standard repo-root browser-proof capture flow.

Process:
1. Read:
   - `.opencode/plans/current-phase.md`
   - `AGENTS.md`
2. Confirm the repo-root browser prerequisites are truthful:
   - run `npm install` from repo root if dependencies are missing
   - run `npx playwright install webkit` from repo root if the helper reports missing WebKit browser binaries
3. Run the repo-root browser smoke helper:
   - `npm run browser:smoke`
4. Treat the helper as the source of truth for the standard browser-proof set:
   - it runs local validation
   - it starts and stops the local preview server at `http://127.0.0.1:4173`
   - it resolves Playwright from the repo root, not from hidden tool-managed paths
   - it captures the standard screenshots into `playwright-artifacts/`
5. Confirm the generated browser-proof set includes:
   - `sessions-screen.png`
   - `task-screen.png`
   - `tool-drawer.png`
   - `offline-baseline.png`
   - `offline-state.png`
   - `offline-recovered.png`
6. Validate at minimum:
   - the Sessions route loads cleanly
   - the Task route loads cleanly
   - no critical primary action depends on horizontal scrolling
   - the shell remains readable on a narrow viewport
   - the tool drawer can open and close without losing task context
   - offline and recovery screenshots were also regenerated by the same repo-owned flow
7. Return a strict PASS or FAIL decision.
8. If FAIL:
   - name the failing condition
   - name the affected route
   - name the smallest likely fix
   - include any helper bootstrap guidance that was required (`npm install` or `npx playwright install webkit`)
9. Do not modify source files unless explicitly asked.

Output format:
- Validation result: PASS|FAIL
- Routes checked
- Screenshots written
- Findings
