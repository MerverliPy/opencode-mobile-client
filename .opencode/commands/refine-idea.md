---
description: Turn a rough product or workflow idea into a bounded backlog-ready candidate
agent: orchestrator
---

Use `docs/ideas/idea-template.md` as the output format.

Goal:
Turn a rough idea into a bounded candidate that can later be added to `.opencode/backlog/candidates.yaml`.

Requirements:
- define the user problem clearly
- define the smallest useful scope
- define explicit non-goals
- name likely primary files or modules
- describe validation direction
- reject vague multi-module rewrites when a smaller slice exists
- preserve iPhone-first and narrow-screen constraints for product work

Return:
- a filled candidate draft
- bounded scope summary
- likely validation command or proof direction
- whether it is ready for backlog intake now or needs refinement first

Do not modify `.opencode/backlog/candidates.yaml` automatically.
